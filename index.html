# 安装使用webpack@4.42.0

  1. 全局安装 npm i webpack -g
  2. 项目安装,在项目根目录中运行，安装成开发依赖 npm i webpack --save-dev (--save-dev可以简写为-D开发依赖， --save可以简写为-S普通依赖)
  3. webpack 4x以上，webpack将命令相关的内容都放到了webpack-cli，所以还需要安装webpack-cli npm install --global webpack-cli
  4. webpack -v 能够显示版本号就是安装好了

NOTE： 当使用npm安装一些全局的软件包时，不知道安装到了什么位置，可以使用命令 npm root -g 进行查询

# PyCharm中运行同一个python程序时选择平行窗口运行

# 问题描述

当我们进行Socket编程时，客户端可能有多个，原则上如果有n个客户端，那么我们就要编辑n个客户端的代码。然而其实我们每个客户端的代码都是相同，如果编辑n遍，将会相当的浪费空间。

# 解决办法

学了并发编程，我们知道，每一个程序重新运行时都是一个新的进程，那么只要我们把客户端运行设置为：重新打开一个平行的窗口运行而不是关闭当前窗口重新运行，就可实现多个客户端的效果。设置方法如下：pycharm工具栏---->run---->Edit Configurations---->Allow parallal run（在右上角的位置），勾上它即代表重新运行时是开一个新的窗口。

# win10 64位安装mysql 5.7

  • 下载5.7 免安装版 2选1

    • https://dev.mysql.com/downloads/mysql/5.7.html#downloads 选择mysql-5.7.30-winx64.zip 64位
    • https://downloads.mysql.com/archives/community/ 选择mysql-5.7.26-winx64 (以这个版本为例)

  • 解压并创建my.ini在根目录 'C:\Users\u243216\Desktop\mysql-5.7.26-winx64'

      [client]
  # 设置mysql客户端默认字符集
  default-character-set=utf8
  [mysqld]
  # 设置3306端口
  port = 3306
  # 设置mysql的安装目录 这块换成自己解压的路径
  basedir=C:\\Users\\u243216\\Desktop\\mysql-5.7.26-winx64
  # 允许最大连接数
  max_connections=200
  # 服务端使用的字符集默认为8比特编码的latin1字符集
  character-set-server=utf8
  # 创建新表时将使用的默认存储引擎
  default-storage-engine=INNODB

    主要修改下basedir后的路径，注意路径中为双反斜杠'\'而非单反斜杠''

  • 以管理员的方式打开cmd命令窗口，并且进入到mysql安装目录的bin目录下 'cd C:\Users\u243216\Desktop\mysql-5.7.26-winx64\bin'

  • 运行 mysqld --initialize 此时应该没有任何提示 mysqld --initialize会帮助你创建data文件夹并初始化mysql, 因此不要自己创建data文件夹

  • 执行 mysqld -install 安装mysql服务

  • 执行 net start mysql 启动服务 第一次进入mysql会由 mysql initialize 初始化一个密码, 在data/*.err这个文件中搜索password会看到生成的密码

  • 注册环境变量与登录 在环境变量的系统变量中的path 新建一个环境变量,值为bin路径 'C:\Users\u243216\Desktop\mysql-5.7.26-winx64\bin' 输入 'mysql -u root -p' 用刚才看到的密码登录

  • 进入后可以用如下命令修改，这里密码改为root 'ALTER USER 'root'@'localhost' IDENTIFIED BY 'root';'

  • 在mysql中运行退出命令quit'mysql> quit'
