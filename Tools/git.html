# 初始化Git仓库

  • 这个仓库会存放git对我们项目代码进行备份的文件
  • 在项目目录右键打开git bash
  • 命令: git init会生成.git目录 即我们的仓库

# 自报家门

  • 就是在git中设置当前使用的用户是谁
  • 每次备份代码到.git目录中都会把当前备份者的信息存储起来
  • 命令:
    • 配置用户名：git config --global user.name "wenliang"
    • 配置邮箱：git config --global user.email "419099632@qq.com"

# 把代码存储到.git仓库中

  • 1.把代码放到仓库门口
    • git add ./readme.md 把指定的文件添加到大门口
    • git add ./ 把当前目录下所有修改的文件添加到大门口
  • 2.把仓储门口的代码放到里面的房间里去
    • git commit -m "这是对这次添加东西的说明"

# 可以一次性把我们修改的代码放到房间里（版本库），不需要先add再commit

  • git commit --all -m "一些说明"
    • --all 表示把所有修改的文件提交到版本库

# 查看当前的状态

  • 可以用来查看当前代码有没有被放到仓储中去
  • 命令：git status

# git中的忽略文件

  • .gitignore,在这个文件中可以设置要忽略的文件或目录
  • 被忽略的文件不会被提交的仓库里去
  • 在.gitignore中可以书写要忽略的文件的路径，以/开头，一行写一个路径，这些路径所对应的文件都会被忽略，不会被提交到仓储中
    • 写法
      • /.idea 会忽略.idea文件
      • /js 会忽略js目录里的所有文件
      • /js/*.js 会忽略js目录里的所有js文件

# 查看日志

  • git log 查看历史提交日志
  • git log --oneline 可以查看简洁版的日志

# 回退到指定的版本

  • git reset --hard Head~0
    • 表示回退到上一次代码提交时的状态
  • git reset --hard Head~1
    • 表示回退到上上一次代码提交时的状态
  • git reset --hard [版本号]
    • 可以通过版本号精确的退回到某一个词
  • git reflog
    • 可以看到每一次切换版本的记录（切换和提交版本的记录），可以看到所有提交的版本号

# 分支

  • 默认是有一个主分支 master

# 创建分支

  • git branch dev
  • 创建了一个dev分支
  • 在刚创建的dev分支里的东西和master分支里的东西是一样的

# 切换分支

  • git checkout dev
  • 切换到指定的分支，这里的切换到名为dev的分支， git branch可以查看当前有哪些分支

# 合并分支

  • git merge dev
  • 合并分支内容，把当前分支与指定的分支（dev）进行合并
  • 当前分支指的是git branch命令输出的前面有*号的分支
  • 合并时如果有冲突，需要手动去处理，处理后还需要再提交一次

# Github

  • https://github.com github不是git是一个网站提供了允许别人通过git上传代码的功能

# 提交代码到github（当作git服务器来用）

  • git push [地址] master
  • 示例 ：git push https://github.com/WenLiang619/git_test.git master
  • 会把当前分支的内容上传到远程的master分支上

# 从github拉取代码到本地

  • git pull [地址] master

  • 示例 ：git pull https://github.com/WenLiang619/git_test.git master

  • 会把远程分支的数据得到：（注意本地要首先初始化一个仓储 git,用 git init）

  • git clone [地址] --depth=1

  • 会得到远程仓库相同的数据，如果多次执行会覆盖本地内容 depth用于指定克隆深度，为1即表示只克隆最近一次commit

# push与pull操作

  • 先pull, 再push 服务器版本有可能与本地版本不一样，所以要先拉取下来合并后再推到服务器上

# 简化push/pull

  • 添加 origin变量git remote add origin https://github.com/WenLiang619/git_test.git
  • 在当前目录下 可以这样push git push origin master 这样pull git pull origin master
  • 执行git push origin -u master 后,git会把当前分支与远程分支做了关联， 下次一push时只要写git push, pull时只要写git pull

  • git status
  • git add ./
  • git commit -m 'second'
  • git push
